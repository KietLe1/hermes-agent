Hermes Agent trên VPS Ubuntu 24.04 ☤

Tự host một Hermes Agent chạy 24/7, dùng Codex (đăng nhập ChatGPT, không API key) làm bộ não, kết nối Discord — cài từ đầu trên Ubuntu 24.04 mới tinh, theo hướng bảo mật trước hết.

⚠ Đọc trước khi bắt đầu — vì sao bảo mật là số 1.
Hermes có tool terminal: nó chạy lệnh shell thật trên VPS. Khi mở qua Discord 24/7, bất kỳ ai bạn cho phép nhắn bot đều có thể khiến nó chạy lệnh. Vì vậy tài liệu này bắt buộc: (1) chạy bằng user thường, KHÔNG phải root; (2) allowlist Discord chỉ mình bạn; (3) khóa SSH + firewall; (4) tùy chọn cô lập terminal bằng Docker. Đừng bỏ qua các bước bảo mật.
Kiến trúc & cổng mạng: Bot Discord kết nối ra ngoài (outbound websocket tới Discord). Hermes không cần mở cổng vào nào để chạy bot. Nhờ vậy firewall có thể đóng gần như toàn bộ inbound — chỉ chừa SSH. (Chỉ khi nào bạn bật Web Dashboard công khai mới cần mở thêm cổng — guide này KHÔNG bật.)

0Chuẩn bị

Bạn cần sẵn:

  • Một VPS Ubuntu 24.04 mới (1 vCPU / 1–2 GB RAM là chạy được; 2 GB thoải mái hơn).
  • IP VPS + cách đăng nhập ban đầu (thường là root + mật khẩu, hoặc key do nhà cung cấp cấp).
  • Máy tính cá nhân của bạn (để mở trình duyệt khi đăng nhập Codex & tạo Discord bot).
  • Tài khoản ChatGPT/Codex (cho bộ não) và tài khoản Discord.
Mọi lệnh bắt đầu bằng $ chạy trên VPS. Lệnh nào chạy trên máy bạn sẽ ghi rõ “trên máy cá nhân”.

1Đăng nhập VPS + cập nhật hệ thống

Trên máy cá nhân, mở terminal (PowerShell/Terminal) và SSH vào VPS (thay IP):

ssh root@YOUR_VPS_IP

Cập nhật toàn bộ gói & cài vài tiện ích cơ bản:

apt update && apt -y upgrade
apt -y install curl git ufw fail2ban unattended-upgrades ca-certificates
timedatectl set-timezone Asia/Ho_Chi_Minh

Nếu kernel được nâng cấp, có thể cần reboot rồi SSH lại.

2Tạo user thường có sudo

Không bao giờ chạy Hermes bằng root. Agent chạy lệnh shell — nếu là root, một lệnh sai (hoặc bị lạm dụng qua Discord) có thể phá cả máy.

Tạo user hermes và cấp quyền sudo:

adduser hermes          # đặt mật khẩu mạnh khi được hỏi
usermod -aG sudo hermes

3Khóa SSH bằng key + tắt mật khẩu/root

Thứ tự rất quan trọng để KHÔNG tự khóa mình ra ngoài: cài key → kiểm tra đăng nhập bằng key ở một cửa sổ MỚI → mới tắt mật khẩu.

3.1 Tạo & nạp khóa SSH

Trên máy cá nhân (nếu chưa có key):

ssh-keygen -t ed25519 -C "hermes-vps"

Chép public key lên VPS cho user hermes (cách dễ nhất, trên máy cá nhân):

ssh-copy-id hermes@YOUR_VPS_IP

Windows không có ssh-copy-id? Làm thủ công: trên VPS chạy mkdir -p /home/hermes/.ssh && nano /home/hermes/.ssh/authorized_keys, dán nội dung file ~/.ssh/id_ed25519.pub từ máy bạn vào, lưu lại, rồi:

chown -R hermes:hermes /home/hermes/.ssh
chmod 700 /home/hermes/.ssh
chmod 600 /home/hermes/.ssh/authorized_keys

3.2 KIỂM TRA đăng nhập bằng key (cửa sổ mới)

Trên máy cá nhân, mở cửa sổ terminal MỚI (giữ cửa sổ root cũ đang mở phòng hờ):

ssh hermes@YOUR_VPS_IP

Vào được mà không hỏi mật khẩu (hoặc chỉ hỏi passphrase của key) → OK, tiếp tục.

3.3 Tắt mật khẩu & đăng nhập root qua SSH

Ảnh cloud Ubuntu 24.04 thường có file đè /etc/ssh/sshd_config.d/50-cloud-init.conf bật lại mật khẩu. Cách chắc ăn: tạo file drop-in tên đứng sau cùng (thắng mọi file khác).

Trên VPS (đang là user hermes, dùng sudo):

sudo tee /etc/ssh/sshd_config.d/99-hardening.conf > /dev/null <<'EOF'
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
KbdInteractiveAuthentication no
EOF
sudo systemctl restart ssh
Đừng đóng cửa sổ root cũ vội. Mở thêm một cửa sổ MỚI và thử ssh hermes@IP lần nữa. Vào được mới đóng cửa sổ root. Từ giờ chỉ đăng nhập bằng user hermes + key.

4Firewall + fail2ban + tự vá bảo mật

4.1 UFW — chỉ chừa SSH

sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status verbose

Bot Discord đi outbound nên không cần mở cổng vào nào khác.

4.2 fail2ban (chống dò mật khẩu SSH)

sudo systemctl enable --now fail2ban
sudo fail2ban-client status sshd

4.3 Tự động cập nhật bản vá bảo mật

sudo dpkg-reconfigure -plow unattended-upgrades   # chọn <Yes>
systemctl status unattended-upgrades --no-pager

5Cài Hermes (dưới user thường)

Đăng nhập VPS bằng user hermes (không phải root) rồi mới cài. Trên Linux, HERMES_HOME = /home/hermes/.hermes.
Thận trọng “curl | bash”: bạn đang chạy script cài từ mạng. Nếu muốn yên tâm, tải về xem trước rồi mới chạy: 
curl -fsSL https://hermes-agent.nousresearch.com/install.sh -o install-hermes.sh
less install-hermes.sh      # xem nội dung
bash install-hermes.sh
Hoặc chạy thẳng một dòng như dưới nếu bạn tin nguồn.
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
source ~/.bashrc
hermes --version
hermes doctor

Installer tự lo Python 3.11, Node, ripgrep, ffmpeg (cài cục bộ trong ~/.hermes, không cần admin). doctor sẽ cảnh báo “chưa login” và “thiếu DISCORD_BOT_TOKEN” — đúng như mong đợi, ta xử lý ở các bước sau.

Muốn chạy bản fork của bạn thay vì bản chính thức? Thay bước này bằng: git clone <fork-url> → cài uv → uv venv venv --python 3.11 → uv pip install -e ".[all,messaging]" --python ./venv/bin/python, rồi dùng ./venv/bin/hermes ở mọi lệnh bên dưới. Vì không cần sửa code, bản chính thức là đủ và đơn giản hơn cho production.

6Đăng nhập Codex (device-code — hợp headless)

Codex dùng luồng device-code: VPS in ra một URL + mã, bạn mở URL trên trình duyệt máy cá nhân để duyệt — không cần trình duyệt trên VPS.

hermes auth add openai-codex
  1. VPS in URL + mã. Mở URL đó trên máy cá nhân, đăng nhập ChatGPT/Codex, nhập mã, xác nhận.
  2. Kết quả: Added openai-codex OAuth credential #1.

Đặt Codex làm model chính & kiểm tra:

hermes config set model.provider openai-codex
hermes config set model.default gpt-5.3-codex   # đổi theo model bạn có; hoặc chạy `hermes model`
hermes auth status openai-codex                 # -> logged in
(Tùy chọn) Gemini qua đăng nhập Google trên VPS headless: luồng Gemini OAuth mở cổng loopback nên cần SSH port-forward. Trên máy cá nhân: ssh -L 8085:localhost:8085 hermes@IP rồi mới chạy hermes auth add google-gemini-cli trên VPS, và mở link nó in ra trên trình duyệt máy bạn. Không bắt buộc — Codex đã đủ làm bộ não.

7Tạo Discord bot + cấu hình .env an toàn

7.1 Tạo bot (trên máy cá nhân, trình duyệt)

  1. Vào discord.com/developers/applications → New Application.
  2. Tab Bot → Reset Token → Copy token (giữ bí mật tuyệt đối).
  3. Vẫn tab Bot → bật MESSAGE CONTENT INTENT (bắt buộc để bot đọc nội dung tin nhắn).
  4. Tab OAuth2 → URL Generator: scopes bot + applications.commands; permissions: Send Messages, Read Message History, Create Public Threads, Add Reactions. Mở URL sinh ra → mời bot vào server của bạn.
  5. Lấy User ID của bạn: Discord → Settings → Advanced → bật Developer Mode; chuột phải tên bạn → Copy User ID.

7.2 Ghi token vào .env (trên VPS)

nano ~/.hermes/.env

Thêm (thay giá trị trong <...>):

DISCORD_BOT_TOKEN=<token-bạn-copy>
DISCORD_ALLOWED_USERS=<user-id-của-bạn>

Khóa quyền đọc file bí mật:

chmod 600 ~/.hermes/.env
Allowlist là lớp phòng thủ chính. Chỉ để DISCORD_ALLOWED_USERS là chính bạn (có thể nhiều ID, cách nhau dấu phẩy). TUYỆT ĐỐI KHÔNG đặt DISCORD_ALLOW_ALL_USERS=true — nếu không, bất kỳ ai cũng có thể bảo bot chạy lệnh shell trên VPS của bạn.

7.3 (Khuyến nghị) siết hành vi Discord

nano ~/.hermes/config.yaml

Thêm khối top-level:

discord:
  require_mention: true      # trong kênh server phải @mention bot mới trả lời
  auto_thread: true
  reactions: true

8Chạy thử gateway (foreground)

Chạy trực tiếp một lần để xem log kết nối:

hermes gateway

Mong đợi log kiểu Discord connected as <tên-bot>. Sang Discord, @mention bot (hoặc DM): bạn đang chạy model nào? → bot trả lời (do Codex). Xong nhấn Ctrl+C để dừng (bước sau sẽ cho chạy nền).

Foreground chỉ để test — đóng SSH là bot tắt. Đừng dừng ở đây; làm tiếp Bước 9 để chạy 24/7.

9Chạy 24/7 bằng systemd

Cách A — để Hermes tự cài service (dễ)

hermes gateway install
hermes gateway status
hermes gateway stop      # khi cần dừng

Cách B — viết unit thủ công (kiểm soát + tự khởi động lại)

Lấy đường dẫn tuyệt đối của hermes:

which hermes     # ví dụ: /home/hermes/.local/bin/hermes

Tạo service (sửa ExecStart theo kết quả trên):

sudo tee /etc/systemd/system/hermes-gateway.service > /dev/null <<'EOF'
[Unit]
Description=Hermes Agent Gateway (Discord)
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=hermes
Group=hermes
WorkingDirectory=/home/hermes
Environment=HERMES_HOME=/home/hermes/.hermes
ExecStart=/home/hermes/.local/bin/hermes gateway
Restart=always
RestartSec=5
# Giới hạn tài nguyên (chống chạy loạn)
MemoryMax=1500M
TasksMax=400

[Install]
WantedBy=multi-user.target
EOF
sudo systemctl daemon-reload
sudo systemctl enable --now hermes-gateway
systemctl status hermes-gateway --no-pager
Tự khởi động lại nếu crash (Restart=always) và tự chạy lại sau khi VPS reboot (enable). Không bật ProtectSystem=strict vì agent cần chạy lệnh thật — lớp cô lập đặt ở user thường + Docker terminal (Bước 10).

10(Nâng cao) Cô lập tool terminal bằng Docker

Đây là nâng cấp bảo mật mạnh nhất: cho tool terminal chạy trong container Docker thay vì trực tiếp trên VPS, để dù bị lạm dụng cũng khó chạm vào hệ thống host.
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker hermes
newgrp docker   # hoặc đăng xuất/đăng nhập lại

Trong ~/.hermes/config.yaml, đặt terminal backend = docker (theo mẫu trong cli-config.yaml.example):

terminal:
  backend: "docker"
  cwd: "/workspace"
  docker_image: "nikolaik/python-nodejs:python3.11-nodejs20"
  docker_run_as_host_user: true
  container_memory: 2048
  container_persistent: true

Khởi động lại service để áp dụng: sudo systemctl restart hermes-gateway.

Đánh đổi: chậm hơn chút khi khởi động container, và một số tác vụ cần cài thêm gói trong image. Nhưng an toàn hơn nhiều cho agent mở ra Internet.

12Checklist bảo mật cuối cùng

  • ☐ Hermes chạy bằng user hermes, KHÔNG phải root (ps -o user= -C python để kiểm tra tiến trình).
  • ☐ SSH: chỉ key, đã tắt PasswordAuthentication & PermitRootLogin.
  • ☐ UFW bật, chỉ chừa OpenSSH; fail2ban đang chạy.
  • ☐ unattended-upgrades bật (tự vá bảo mật).
  • ☐ ~/.hermes/.env có chmod 600; không có DISCORD_ALLOW_ALL_USERS=true.
  • ☐ DISCORD_ALLOWED_USERS chỉ là (các) ID của bạn; require_mention: true.
  • ☐ service enable (tự chạy sau reboot) + Restart=always.
  • ☐ (Khuyến nghị) terminal chạy trong Docker (Bước 10).
  • ☐ Đã backup ~/.hermes.
